# Web

修仙 Mini Web 的前端。

## 当前实现

无构建依赖的原生 HTML/CSS/JS 单页,由 `backend/app.py` 直接静态托管。第一屏直接是 Telegram 消息聊天界面,不是营销页或设置页。

文件:

- `index.html` - 游戏客户端骨架(官方定时和常用入口侧栏 / 消息流 / 工具中心 / 发送栏)。
- `static/app.js` - 启动入口和仍未拆出的主工作台逻辑。
- `static/state.js` / `static/api.js` / `static/constants.js` - 全局状态、HTTP 封装和固定展示配置。
- `static/ui/` - 通用 UI helper,例如弹层和格式化。
- `static/views/` - 已拆出的独立弹层/面板,例如日志、草稿箱、健康、背包、通知、过滤、情报、虚天攻略和苍坤攻略。
- `static/styles.css` - 全部样式。

## UI 边界

- 主消息流以 Telegram 原文为第一优先级,按时间分日期分组,长文本默认折叠并保留关键段。
- 解析得到的标题、标签、动作数量作为轻量提示放在原文下方,不替代原文。
- 风险类消息用温和警告色 + 左侧色条 + 「需要关注」徽章,不做成报错页。
- 频道过滤放在消息流顶部,支持横向滚动和合并展示;主区标题显示当前激活频道。
- 工具中心收纳账号、身份、角色状态、过滤设置和通知设置;重点流、副本、官方定时、发送栏不放进二级。
- 消息详情围绕「Telegram 原文 + 增强字段 + 动作草稿」三块。
- 动作只生成 outbox 草稿,从不直接发送;同一气泡里有「复制命令」「确认入队」「查看发送计划」三个按钮。
- 主侧栏「官方定时」支持查看当前批次、刷新、新建、预览、创建、取消、删除和拉取 Telegram 端状态对账;多身份排班会按偏移自动错开。
- 主侧栏「常用」放副本状态、资源统计、库存/转移、草稿箱和记录入口,保证常用主流程不进二级。
- 「副本状态」只读取消息箱近期副本卡片,按副本 ID / 名称汇总开房、加入结果、失败、解散和推进消息;不独立维护玩法状态,也不自动加入。
- 「过滤设置」在工具中心里,提供关注关键词预设和排除规则预览;排除规则只把普通重点噪音转入归档,不会压掉 @我、风险和动作卡。
- 前端在页面可见时做 5 秒增量轮询,把 listener 已入库的新消息合并进当前消息流;手动「刷新」仍可做完整重载。

## 不做

- 不引入大型前端框架,继续原生 JS/CSS。
- 不做自动发送、自动重试、自动补发。
- UI 文案使用简体中文,不使用繁体。
