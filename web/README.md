# Web

修仙 Mini Web 的前端。

## 当前实现

无构建依赖的原生 HTML/CSS/JS 单页,由 `backend/app.py` 直接静态托管。第一屏直接是 Telegram 消息聊天界面,不是营销页或设置页。

文件:

- `index.html` - 三栏骨架(账号/频道侧栏 / 消息流 / 详情)。
- `static/app.js` - 全部交互逻辑。
- `static/styles.css` - 全部样式。

## UI 边界

- 主消息流以 Telegram 原文为第一优先级,按时间分日期分组,长文本默认折叠并保留关键段。
- 解析得到的标题、标签、动作数量作为轻量提示放在原文下方,不替代原文。
- 风险类消息用温和警告色 + 左侧色条 + 「需要关注」徽章,不做成报错页。
- 频道过滤左侧多选,合并展示;主区标题显示当前激活频道。
- 右侧详情围绕「Telegram 原文 + 增强字段 + 动作草稿」三块。
- 动作只生成 outbox 草稿,从不直接发送;同一气泡里有「复制命令」「确认入队」「查看发送计划」三个按钮。
- 顶栏「草稿箱」按钮在右侧详情列出已入队、等待人工确认的草稿,可复制、查看原消息或删除。
- 顶栏「官方定时」当前是占位说明,后续接 Telegram 官方定时消息的 create/list/delete。

## 不做

- 不引入大型前端框架,继续原生 JS/CSS。
- 不做自动发送、自动重试、自动补发。
- 不在前端轮询消息,刷新只走右上角刷新按钮(或重新打开页面)。
- UI 文案使用简体中文,不使用繁体。
