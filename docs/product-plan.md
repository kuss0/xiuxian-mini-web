# Product Plan

## 一句话定位

为 Telegram 修仙群提供一个轻量 Web 图形辅助界面，把刷屏聊天流整理成可阅读、可筛选、可操作的游戏信息面板。

## 用户问题

- 群消息密度高，玩家聊天、系统公告、bot 回复混在一起。
- 很多玩法依赖复制指令、识别 CD、查看状态，纯聊天窗口操作成本高。
- 繁体用户对简体命令复制和输入不方便。
- 自动脚本存在封号风险，用户需要更像“客户端辅助界面”的工具，而不是后台自动化傀儡。

## 产品原则

1. 游戏本体仍在 Telegram。
2. Mini Web 只做阅读整理、人工确认、官方定时。
3. 所有发送动作必须可解释、可追踪、可取消。
4. 默认不追求极限收益，优先降低异常行为和误触发风险。
5. UI 面向玩家语言，底层保留原始消息和命令。

## 信息分流

底层消息标签：

- `world`：世界聊天、普通玩家发言。
- `system`：系统公告、全服事件。
- `mine`：当前角色相关消息。
- `training`：修炼、闭关、元婴、第二元神。
- `dungeon`：副本开启、加入、队伍状态。
- `resource`：储物袋、交易、资源转移。
- `home`：洞府、灵树、小世界、侍妾、法宝、灵兽。
- `risk`：举报、自证、禁言、虚弱、封禁风险。
- `log`：本工具自己的操作记录。

用户视图：

- 我的修仙：只看与当前角色有关的重点消息。
- 游戏大厅：世界聊天、系统公告、公共事件。
- 事件中心：副本、交易、风险、CD、可响应事件。
- 操作台：根据消息生成可复制或可确认发送的动作卡。

## MVP

第一阶段只做最小闭环：

1. 读取指定 Telegram 群最近消息。
2. 按规则给消息打标签。
3. Web 中展示场景 Tab 和频道过滤器。
4. 点击消息显示摘要、原文、识别出的角色和玩法。
5. 从消息生成可复制命令。
6. 支持用户确认后发送命令。
7. 支持创建、查看、删除 Telegram 官方定时消息。

## 暂不做

- 自动挂机调度。
- 自动刷状态。
- 自动补发。
- 自动链式命令。
- 自动答题或防举报 AI。
- 移动端原生 App。

## 后续可扩展

- 自定义解析规则。
- 自定义频道和场景。
- 多账号身份管理。
- 定时排班模板。
- 常用玩法预设，例如深度闭关、抚摸法宝、温养器灵。
- 与现有修仙脚本共享只读状态，但不共享自动发送链路。
